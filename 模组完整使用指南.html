简介

什么是数据包配置模组？

数据包配置模组是一个为 Minecraft Forge 1.20.1 开发的工具模组，旨在为数据包开发者提供便捷的图形化配置界面。通过集成 Cloth Config API，模组允许玩家在游戏中直接修改数据包的配置参数，而无需手动编辑 JSON 文件或重启游戏。

主要特点

🎨 图形化配置界面

使用 Cloth Config 提供友好的用户界面

🔧 10+ 种配置类型

支持滑块、输入框、下拉菜单、颜色选择器等多种输入方式

📝 实时修改

配置修改立即生效，无需重启游戏

🌍 多语言支持

完整的国际化支持，支持简体中文和英语

📦 批量修改

一个配置项可同时修改多个文件位置

🔄 嵌套路径支持

支持 JSON 嵌套路径（如 terrain/scale）

💾 ZIP 数据包支持

支持 ZIP 格式的数据包

🛡️ 安全可靠

自动备份、临时文件机制、完善的错误处理

核心功能

1. 配置管理

模组提供了完整的配置管理功能：

  • 自动扫描: 自动扫描数据包文件夹中的所有数据包
  • 配置加载: 自动解析每个数据包的配置定义
  • 界面构建: 动态为每个数据包生成配置界面
  • 保存应用: 将配置修改应用到数据包文件中

2. 支持的配置类型

类型标识 数据类型 UI 组件 功能说明
sliderInteger整数滑块拖动调整整数值
double_sliderDouble浮点滑块拖动调整浮点数值
intInteger整数输入框直接输入整数值
doubleDouble浮点输入框直接输入浮点数值
booleanBoolean开关切换器启用/禁用开关
stringString文本输入框文本输入
enumString下拉菜单从预设选项中选择

colorInteger (HEX)颜色选择器RGB/ARGB 颜色选择
dropdown_menuString选择器组件自定义下拉菜单

3. 数据包格式支持

  • ZIP 格式: 自动解压、修改、重新打包
  • 自动备份: ZIP 文件修改前自动备份

文件修改机制

JSON 文件修改

// 原始文件
{ "terrain": { "scale": 1.0 }}

// 配置: key = "terrain/scale", value = 2.0

// 结果文件
{ "terrain": { "scale": 2.0 }}

函数文件修改

# 原始文件
set debug_enabled false

# 配置: key = "debug_enabled", value = true

# 结果文件
set debug_enabled true

安装与配置

1. 安装要求

  • Minecraft 版本: 1.20.1
  • Forge 版本: 47.2.0 或更高
  • Cloth Config 模组: 最新版本

2. 安装步骤

  1. 下载模组 JAR 文件
  2. 将文件放入 mods 文件夹
  3. 确保 Cloth Config 模组也已安装
  4. 启动游戏

3. 配置文件位置

文件名 路径 用途
datapack-config-mod.toml config/datapack-config-mod.toml Forge 配置文件
datapack-settings.json config/datapack-config-mod/datapack-settings.json 数据包设置存储
datapack_config.json 数据包根目录 数据包配置定义

4. 数据包文件夹路径

默认路径: config/datapack-config-mod

可在 Forge 配置文件中修改:

[general]
# 数据包文件夹路径
datapack_folder_path = "config/datapack-config-mod"

配置参数完整说明

配置项核心参数

每个配置项都需要定义以下核心参数：

  1. type (必需)

    类型: String
    说明: 配置项的数据类型，决定 UI 组件和验证规则
    可选值 : slider , double_slider , int , double , boolean , string , enum , enum_selector , color , dropdown_menu
    示例:
    "type": "slider"

  2. default (必需)

    类型: String
    说明: 配置项的默认值，必须为字符串格式
    注意: 所有默认值都必须用引号包裹，即使是数字
    示例:
    "default ": "50 " // 正确 ✓
    "default ": 50 // 错误 ✗
    "default ": "true " // 正确 ✓
    "default ": true // 错误 ✗
    "default ": "#FFFF5733 " // RGB格式 正确 ✓
    "default ": "#FF5733 " // ARGB格式 正确 ✓

  3. key (必需或可选)

    类型: String
    说明: 配置项在目标文件中的标识符
    使用场景:
    JSON 文件: JSON 路径（支持嵌套，用 / 分隔）
    函数文件: 命令中的参数名
    文本文件: 配置的标识符
    必需条件: 当使用 filePath 时必需，当使用 locations 时不需要
    示例:
    // 简单键
    "key": "spawn_rate"
    // 嵌套 JSON 路径
    "key": "terrain/scale"
    // 深层嵌套
    "key": "argument1/argument/xz_scale"
    注意事项:
    嵌套路径使用 / 作为分隔符
    路径中的每一级都对应 JSON 中的一个对象层级
    如果路径中的某一层级不存在，模组会自动创建

  4. filePath (必需或可选)

    类型: String
    说明: 目标文件在数据包中的路径
    必需条件: 当使用 key 时必需，当使用 locations 时不需要
    路径格式:
    必须以 data/ 开头
    相对于数据包根目录的路径
    支持 JSON 文件和 .mcfunction 文件
    示例:
    // JSON 配置文件
    "filePath": "data/example/config/settings.json"
    // 函数文件
    "filePath": "data/example/functions/setup.mcfunction"
    // 深层路径
    "filePath": "data/example/worldgen/biome/forest.json"
    支持的文件类型:
    .json: JSON 配置文件，支持嵌套路径
    .mcfunction: 函数文件，支持命令参数修改
    .txt: 文本文件（部分支持）
    注意事项:
    路径区分大小写
    文件必须存在，否则配置无法应用
    建议使用相对路径，不要使用绝对路径

  5. locations (可选)

    类型: Array<Object>
    说明: 批量修改配置，一个配置项可同时修改多个文件位置
    使用场景: 当需要将一个配置应用到多个文件或多个位置时使用
    结构:
    "locations": [ { "filePath": "String", // 目标文件路径 "key": "String", // 配置键 "description": "String" // 描述（可选） }]
    示例:

    { "debug_mode ": { 
  "type ": "boolean ", 
  "default ": "false ", 
  "locations ": [ 
    { 
      "filePath ": "data/example/config/debug.json ", 
      "key ": "debug.enabled ", 
      "description ": "主调试配置 " 
    }, 
    { 
      "filePath ": "data/example/config/main.json ", 
      "key ": "debug_mode ", 
      "description ": "主配置中的调试开关 " 
    }, 
    { 
      "filePath ": "data/example/functions/debug.mcfunction ", 
      "key ": "debug_enabled ", 
      "description ": "调试函数控制 " 
    } 
  ] 
}}
    与 filePath + key 的区别:
    filePath + key: 单个位置修改，简单配置
    locations: 多个位置修改，适用于全局开关、同步参数等场景
    注意事项:
    locations 和 filePath+key 不能同时使用
    数组中的每个对象都必须有 filePath 和 key
    description 是可选的，用于调试和日志记录

  6. name (可选)

    类型: Object (多语言映射)
    说明: 配置项在界面中显示的标题
    结构:
    "name": { "zh_cn": "简体中文标题", "en_us": "English Title"}
    示例:
    "name": { "zh_cn": "生物生成率", "en_us": "Mob Spawn Rate"}
    优先级: 高于 displayNameKey
    如果不配置: 显示配置项的键名（JSON 中的 key）

  7. description (可选)

    类型: Object (多语言映射)
    说明: 配置项下方的说明文本
    显示位置: 配置项名称下方，使用灰色小字体显示
    结构:
    "description": { "zh_cn": "简体中文描述", "en_us": "English Description"}
    示例:
    "description": { "zh_cn": "控制生物的生成频率，数值越大生成越快", "en_us": "Controls mob spawn frequency, higher values spawn faster"}
    与 tooltip 的区别:
    description: 始终显示在配置项下方
    tooltip: 鼠标悬停时显示

  8. tooltip (可选)

    类型: Object (多语言映射)
    说明: 鼠标悬停时显示的提示文本
    触发方式: 鼠标悬停在配置项上
    结构:
    "tooltip": { "zh_cn": "简体中文提示", "en_us": "English Tooltip"}
    示例:
    "tooltip": { "zh_cn": "拖动滑块调整数值，或点击输入精确值", "en_us": "Drag slider to adjust value, or click to input precise value"}
    与 description 的区别:
    tooltip: 悬停时显示，适合详细说明
    description: 始终显示，适合简短说明
    如果不配置: 不显示提示

  9. category (可选)

    类型: String
    说明: 配置项所属的分类，支持多级分类
    格式: 使用 / 分隔多级分类
    示例:
    // 单级分类
    "category": "game_settings"
    // 多级分类
    "category": "game_settings/difficulty_settings"
    // 三级分类
    "category": "game_settings/difficulty_settings/combat"
    界面显示效果:
    游戏设置
    └─ 难度设置
        └─ 战斗难度等级 [配置项]
    如果不配置: 配置项显示在数据包根分类下

  10. categoryDisplayNames (可选)

    类型: Object (多语言映射的多级映射)
    说明: 自定义分类的显示名称
    结构:
    "categoryDisplayNames ": { "zh_cn ": { "game_settings ": "游戏设置 ", "difficulty_settings ": "难度设置 " }, "en_us ": { "game_settings ": "Game Settings ", "difficulty_settings ": "Difficulty Settings " }}
    使用场景: 当分类的键名不够友好时，使用这个参数自定义显示名称
    如果不配置: 显示分类的键名（category 中的字符串）

  11. displayNameKey (可选)

    类型: String
    说明: 引用数据包语言文件中的显示名称键
    使用场景: 当数据包已经有语言文件时，避免重复定义文本
    示例:
    "displayNameKey": "datapack.example.config.spawn_rate.name"
    对应语言文件 (assets/example/lang/zh_cn.json):
    { "datapack.example.config.spawn_rate.name": "生物生成率"}
    优先级: 低于 name

  12. descriptionKey (可选)

    类型: String
    说明: 引用数据包语言文件中的描述文本键
    使用场景: 当数据包已经有语言文件时，避免重复定义文本
    示例:
    "descriptionKey": "datapack.example.config.spawn_rate.desc"
    对应语言文件 (assets/example/lang/zh_cn.json):
    { "datapack.example.config.spawn_rate.desc": "控制生物的生成频率"}
    优先级: 低于 description

  13. tooltipKey (可选)

    类型: String
    说明: 引用数据包语言文件中的提示文本键
    使用场景: 当数据包已经有语言文件时，避免重复定义文本
    示例:
    "tooltipKey": "datapack.example.config.spawn_rate.tooltip"
    对应语言文件 (assets/example/lang/zh_cn.json):
    { "datapack.example.config.spawn_rate.tooltip": "数值范围: 0-100"}
    优先级: 低于 tooltip

特定类型参数

  1. min (slider, double_slider, int, double 必需)

    类型: Number
    说明: 数值的最小值（包含）
    示例:
    "min": 0
    "min": 0.1

  2. max (slider, double_slider, int, double 必需)

    类型: Number
    说明: 数值的最大值（包含）
    示例:
    "max": 100
    "max": 5.0

  3. step (slider, double_slider 可选)

    类型: Number
    说明: 滑块拖动的步长
    默认值: 1 (slider) 或 0.1 (double_slider)
    示例:
    "step": 1 // 整数滑块，每次变化 1
    "step": 0.1 // 浮点滑块，每次变化 0.1

  4. enumValues (enum, enum_selector, dropdown_menu 必需)

    类型: Array<String>
    说明: 枚举类型的可选值列表
    示例:
    "enumValues": ["easy", "normal", "hard", "expert"]
    注意事项:
    数组中的值必须是字符串
    这些值会存储在配置中（非显示名称）
    建议使用中划线或下划线命名：easy_mode、easy-mode

  5. enumDisplayNames (enum, enum_selector 可选)

    类型: Object (多语言映射)
    说明: 枚举值的多语言显示名称
    结构:
    "enumDisplayNames ": { "zh_cn ": { "easy ": "简单 ", "normal ": "普通 " }, "en_us ": { "easy ": "Easy ", "normal ": "Normal " } }
    如果不配置: 显示枚举值本身（如 easy, normal）

参数组合规则

基本规则

  • 每个配置项必须包含: type 和 default
  • 位置指定方式（二选一）:
    • 方式 A: filePath + key
    • 方式 B: locations 数组
  • 文本相关（可选）: name, description, tooltip 等

特殊规则

  • slider, double_slider, int, double: 必须包含 min 和 max
  • enum, enum_selector, dropdown_menu: 必须包含 enumValues
  • enum, enum_selector: enumDisplayNames 为可选
  • locations 存在时，filePath 和 key 不应再出现在同级

minNumber✓最小值"min": 0
maxNumber✓最大值"max": 100
stepNumber✗步长，默认为 1"step": 5
filePathString✓目标文件路径"filePath": "data/example/config.json"
keyString✓JSON 键或函数参数名"key": "mob_spawn_rate"

{ "mob_spawn_rate ": {

  "default ": "50 ",

  "max ": 100, 
  "step ": 1,

高级功能

1. 批量修改 (Locations)

使用 locations 参数，可以让一个配置项同时修改多个文件中的不同位置。这对于统一管理全局开关或同步参数非常有用。

应用场景

  • 一个全局的“调试模式”开关，同时控制多个配置文件中的相关参数。
  • 一个“世界生成规模”参数，同时影响地形、生物群系、矿物等多个 JSON 文件。

示例

{ "global_debug ": { 
  "type ": "boolean ", 
  "default ": "false ", 
  "locations ": [ 
    { 
      "filePath ": "data/example/config/debug_main.json ", 
      "key ": "debug/enabled " 
    }, 
    { 
      "filePath ": "data/example/config/debug_features.json ", 
      "key ": "feature_debug " 
    }, 
    { 
      "filePath ": "data/example/functions/toggle_debug.mcfunction ", 
      "key ": "debug_toggle " 
    } 
  ] 
}}

当用户将 global_debug 设置为 true 时，模组会自动将三个不同文件中的 debug/enabled、feature_debug 和 debug_toggle 都设置为 true。

2. 嵌套 JSON 路径 (Nested Paths)

通过在 key 中使用 / 分隔符，可以直接访问和修改 JSON 文件中的深层嵌套对象。

示例

目标 JSON 文件 (data/example/config.json):

{ "worldgen ": { "terrain ": { "xz_scale ": 1.0, "y_scale ": 1.0 }, "biomes ": { "ocean_depth ": 16 } } }

对应的配置项:

{ "xz_scale ": { 
  "type ": "double ", 
  "default ": "1.0 ", 
  "min ": 0.1, 
  "max ": 3.0, 
  "filePath ": "data/example/config.json ", 
  "key ": "worldgen/terrain/xz_scale " 
}, 
"ocean_depth ": { 
  "type ": "int ", 
  "default ": "16 ", 
  "min ": 8, 
  "max ": 32, 
  "filePath ": "data/example/config.json ", 
  "key ": "worldgen/biomes/ocean_depth " 
}}

3. 分类与子分类 (Categories)

使用 category 参数可以将配置项组织到不同的分类和子分类中，使界面更加清晰。

示例

{ "spawn_rate ": { 
  "type ": "slider ", 
  "default ": "50 ", 
  "min ": 0, 
  "max ": 100, 
  "filePath ": "data/example/config.json ", 
  "key ": "mobs/spawn_rate ", 
  "category ": "game_settings/mob_spawning " 
}}

这将在配置界面中创建如下结构:

├── 数据包名称
│   ├── 启用此数据包
│   └── 游戏设置 (game_settings)
│       └── 怪物生成 (mob_spawning)
│           └── 怪物生成率 (spawn_rate) [滑块]

配合 categoryDisplayNames 可以自定义分类名称显示:

"categoryDisplayNames ": { 
  "zh_cn ": { 
    "game_settings ": "游戏设置 ", 
    "mob_spawning ": "怪物生成 " 
  } 
}

4. ZIP 数据包支持

模组原生支持 ZIP 格式的 Minecraft 数据包。当检测到 ZIP 文件时，会执行以下操作：

  1. 将 ZIP 文件解压到一个临时文件夹。
  2. 对解压后的文件进行配置修改。
  3. 将修改后的文件重新打包成 ZIP 文件。
  4. 替换原始 ZIP 文件。

在此过程中，原始 ZIP 文件会被自动备份为 original_file.zip.backup，以防修改失败。

5. 自动备份

为了保证数据安全，模组在修改任何文件前都会自动创建备份：

  • JSON 文件: 创建同名的 .backup 文件。
  • ZIP 文件: 创建同名的 .backup 文件。
  • 函数文件: 创建同名的 .backup 文件。

如果配置应用失败，可以手动恢复备份文件。

本地化支持

模组支持完整的多语言本地化，包括配置项名称、描述、提示以及分类名称。支持的语言包括简体中文 (zh_cn) 和英语 (en_us)。

本地化方式

有四种方式可以为配置项提供本地化文本，优先级从高到低排列：

  1. 内联本地化 (Inline Localization): 在 datapack_config.json 中直接定义多语言映射。优先级最高。
  2. 引用数据包语言文件 (Reference Datapack Lang File): 通过 displayNameKey 等键引用数据包自身的 lang 文件。优先级第二。
  3. 引用模组语言文件 (Reference Mod Lang File): 通过 displayNameKey 等键引用模组的 lang 文件。优先级第三。
  4. 显示键名 (Display Key Name): 如果以上方式都未定义，则直接显示 JSON 中的键名。优先级最低。

    "zh_cn ": "怪物生成率 ", 
    "en_us ": "Mob Spawn Rate " 
  }, 
  "description ": { 
    "zh_cn ": "控制怪物的生成频率，数值越大生成越快 ", 
    "en_us ": "Controls the frequency of monster spawning. Higher values mean faster spawning. " 
  }, 
  "tooltip ": { 
    "zh_cn ": "调整此值会影响服务器性能 ", 
    "en_us ": "Adjusting this value may affect server performance " 
  }, 
  "category ": "game_settings/mob_spawning ", 
  "categoryDisplayNames ": { 
    "zh_cn ": { 
      "game_settings ": "游戏设置 ", 
      "mob_spawning ": "怪物生成 "

性能优化

1. 线程安全

所有文件读写操作都在独立的异步线程中进行，不会阻塞主线程或游戏渲染，保证游戏流畅运行。

2. 延迟保存

配置修改后不会立即写入文件，而是等到用户关闭配置界面时统一保存，减少磁盘 I/O 操作。

3. 缓存机制

模组会缓存已扫描的数据包信息和配置定义，在同一游戏会话中重复打开配置界面时速度更快。

4. ZIP 文件优化

对于 ZIP 文件，只解压、修改、重新打包必要的部分，而不是整个文件，提高大文件处理效率。

常见问题

Q1: 模组安装后没有反应？

A: 请检查是否正确安装了 Cloth Config API 模组。此模组是本模组的前置依赖。如果缺少前置，模组将无法正常加载。

Q2: 配置修改后没有生效？

A: 请确保：

  1. 数据包已放置在 config/datapack-config-mod 文件夹中。
  2. 数据包中包含正确格式的 datapack_config.json 文件。
  3. 目标文件（如 filePath 指定的文件）存在于数据包中。
  4. 修改后点击了配置界面的“完成”按钮。

Q3: JSON 嵌套路径不起作用？

A: 请检查 key 的格式。嵌套路径必须使用正斜杠 / 分隔，例如 "key": "worldgen/terrain/scale"。不要使用点号或其他字符。

Q4: 配置项的 default 值报错？

A: 所有的 default 值都必须是字符串格式，即使它代表的是一个数字或布尔值。例如，使用 "default": "50" 而不是 "default": 50。

Q5: ZIP 数据包修改失败？

A: 请检查原始 ZIP 文件是否损坏。修改失败时，原始文件会保留为 filename.zip.backup，可以尝试恢复。同时，检查目标文件路径在 ZIP 内部是否存在。

Q6: 本地化文本不显示？

A: 请确认本地化键名格式正确（如 zh_cn 而不是 zh_CN），并且对应的多语言对象结构正确。内联本地化优先级高于语言文件引用。

Q7: locations 和 filePath/key 可以同时使用吗？

A: 不可以。这两种方式是互斥的。如果定义了 locations，则不应再出现同级的 filePath 和 key。反之亦然。

Q8: 如何为颜色选择器设置默认值？

A: 颜色值必须以 # 开头。支持 RGB (6位) 和 ARGB (8位) 格式。

  • RGB: "default": "#FF5733" (红色)
  • ARGB: "default": "#FFFF5733" (带透明度的红色)
