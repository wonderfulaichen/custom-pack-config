# 2026-05-27

## EpicTerrain 数据包配置界面实现

### 执行过程（含方案迭代）
1. **解析文档** — 从 MHT/PDF/图片中提取 8 个分类参数
2. ❌ 尝试改数据包 JSON 数组为对象 → Minecraft 原版引擎报 `Not a json array`
3. ✅ **最终方案**：改 Mod 代码 + 保持数据包原始格式

### 修改内容

#### A. DatapackFileModifier.java（三个版本：1.20.1/1.21.10/1.21.11）
- 在 `modifyJsonFile` 的路径导航中增加数字数组索引支持
- 当路径段（如 `"0"`、`"1"`）匹配 `\\d+` 时，自动转为 `array.get(index)` 而非 `object.get(key)`
- 新增 `createJsonValue()` 方法用于设置数组元素值
- 新增 import: `JsonArray`, `JsonElement`, `JsonPrimitive`

key 路径示例：`argument/spline/points/1/location` 中 `1` 表示数组索引

#### B. datapack_config.json
- 22 个配置项，按文档 8 个分类组织
- 世界尺度(2)、河流动性(4)、海洋深度(3)、山体高度(3)
- 洞穴高度(4)、群系参数(1)、群系尺度(2)、湖流生成(3)
- key 路径使用数字数组索引（如 `points/1/location`）
- 批量修改：world_scale_xz（c/1.json + c/1a.json）、stream_water_count（水面+冰面）

### 交付文件
- `epicterrain_extracted/datapack_config.json` — 配置文件
- `epicterrain_compatible1.20_configured.zip` — 带配置的完整数据包
- 原始备份：`epicterrain_compatible1.20_original_backup.zip`
