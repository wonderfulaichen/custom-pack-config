## 简介

### 什么是数据包配置模组？

数据包配置模组是一个为 Minecraft Forge 1.20.1 开发的工具模组，旨在为数据包开发者提供便捷的图形化配置界面。通过集成 Cloth Config API，模组允许玩家在游戏中直接修改数据包的配置参数，而无需手动编辑 JSON 文件或重启游戏。

### 主要特点

- **🎨 图形化配置界面**: 使用 Cloth Config 提供友好的用户界面
- **🔧 10+ 种配置类型**: 支持滑块、输入框、下拉菜单、颜色选择器等多种输入方式
- **📝 实时修改**: 配置修改立即生效，无需重启游戏
- **🌍 多语言支持**: 完整的国际化支持，支持简体中文和英语
- **📦 批量修改**: 一个配置项可同时修改多个文件位置
- **🔄 嵌套路径支持**: 支持 JSON 嵌套路径（如 `terrain/scale`）
- **💾 ZIP 数据包支持**: 支持 ZIP 格式的数据包
- **🛡️ 安全可靠**: 自动备份、临时文件机制、完善的错误处理

## 核心功能

### 1. 配置管理

模组提供了完整的配置管理功能：

- **自动扫描**: 自动扫描数据包文件夹中的所有数据包
- **配置加载**: 自动解析每个数据包的配置定义
- **界面构建**: 动态为每个数据包生成配置界面
- **保存应用**: 将配置修改应用到数据包文件中

### 2. 支持的配置类型

| 类型标识 | 数据类型 | UI 组件 | 功能说明 |
|---------|---------|---------|---------|
| `slider` | Integer | 整数滑块 | 拖动调整整数值 |
| `double_slider` | Double | 浮点滑块 | 拖动调整浮点数值 |
| `int` | Integer | 整数输入框 | 直接输入整数值 |
| `double` | Double | 浮点输入框 | 直接输入浮点数值 |
| `boolean` | Boolean | 开关切换器 | 启用/禁用开关 |
| `string` | String | 文本输入框 | 文本输入 |
| `enum` | String | 下拉菜单 | 从预设选项中选择 |
| `enum_selector` | String | 点击切换按钮 | 循环切换选项 |
| `color` | Integer (HEX) | 颜色选择器 | RGB/ARGB 颜色选择 |
| `dropdown_menu` | String | 选择器组件 | 自定义下拉菜单 |

### 3. 数据包格式支持

- **ZIP 格式**: 自动解压、修改、重新打包
- **自动备份**: ZIP 文件修改前自动备份

### 配置流程

#### 1. 初始化阶段

```
游戏启动
  ↓
注册服务器事件
  ↓
注册配置事件
  ↓
创建数据包文件夹
  ↓
扫描并加载数据包
```

#### 2. 配置界面构建阶段

```
打开配置界面
  ↓
扫描数据包文件夹
  ↓
解析每个数据包的配置
  ↓
为每个数据包创建独立分类
  ↓
添加启用/禁用开关
  ↓
按分类组织配置项
  ↓
根据类型创建 UI 组件
```

#### 3. 配置保存与应用阶段

```
用户点击"完成"
  ↓
保存配置到 datapack-settings.json
  ↓
获取数据包文件夹路径
  ↓
遍历所有数据包设置
  ↓
应用配置到数据包文件
  ↓
ZIP 格式: 解压 → 修改 → 重新打包
```

### 文件修改机制

#### JSON 文件修改

```javascript
// 原始文件
{
  "terrain": {
    "scale": 1.0
  }
}

// 配置: key = "terrain/scale", value = 2.0
// 结果文件
{
  "terrain": {
    "scale": 2.0
  }
}
```

#### 函数文件修改

```mcfunction
# 原始文件
set debug_enabled false

# 配置: key = "debug_enabled", value = true
# 结果文件
set debug_enabled true
```

## 安装与配置

### 1. 安装要求

- Minecraft 版本: 1.20.1
- Forge 版本: 47.2.0 或更高
- Cloth Config 模组: 最新版本

### 2. 安装步骤

1. 下载模组 JAR 文件
2. 将文件放入 `mods` 文件夹
3. 确保 Cloth Config 模组也已安装
4. 启动游戏

### 3. 配置文件位置

| 文件名 | 路径 | 用途 |
|-------|------|-----|
| `datapack-config-mod.toml` | `config/datapack-config-mod.toml` | Forge 配置文件 |
| `datapack-settings.json` | `config/datapack-config-mod/datapack-settings.json` | 数据包设置存储 |
| `datapack_config.json` | 数据包根目录 | 数据包配置定义 |

### 4. 数据包文件夹路径

默认路径: `config/datapack-config-mod`

可在 Forge 配置文件中修改:

```toml
[general]
# 数据包文件夹路径
datapack_folder_path = "config/datapack-config-mod"
```

## 配置参数完整说明

### 配置项核心参数

每个配置项都需要定义以下核心参数：

#### 1. `type` (必需)

**类型**: `String`

**说明**: 配置项的数据类型，决定 UI 组件和验证规则

**可选值**: `slider`, `double_slider`, `int`, `double`, `boolean`, `string`, `enum`, `enum_selector`, `color`, `dropdown_menu`

**示例**:
```json
"type": "slider"
```

---

#### 2. `default` (必需)

**类型**: `String`

**说明**: 配置项的默认值，必须为字符串格式

**注意**: 所有默认值都必须用引号包裹，即使是数字

**示例**:
```json
"default": "50"      // 正确 ✓
"default": 50         // 错误 ✗
"default": "true"    // 正确 ✓
"default": true       // 错误 ✗
"default": "#FFFF5733" // RGB格式 正确 ✓
"default": "#FF5733"   // ARGB格式 正确 ✓
```

---

#### 3. `key` (必需或可选)

**类型**: `String`

**说明**: 配置项在目标文件中的标识符

**使用场景**:
- JSON 文件: JSON 路径（支持嵌套，用 `/` 分隔）
- 函数文件: 命令中的参数名
- 文本文件: 配置的标识符

**必需条件**: 当使用 `filePath` 时必需，当使用 `locations` 时不需要

**示例**:
```json
// 简单键
"key": "spawn_rate"

// 嵌套 JSON 路径
"key": "terrain/scale"

// 深层嵌套
"key": "argument1/argument/xz_scale"
```

**注意事项**:
- 嵌套路径使用 `/` 作为分隔符
- 路径中的每一级都对应 JSON 中的一个对象层级
- 如果路径中的某一层级不存在，模组会自动创建

---

#### 4. `filePath` (必需或可选)

**类型**: `String`

**说明**: 目标文件在数据包中的路径

**必需条件**: 当使用 `key` 时必需，当使用 `locations` 时不需要

**路径格式**:
- 必须以 `data/` 开头
- 相对于数据包根目录的路径
- 支持 JSON 文件和 `.mcfunction` 文件

**示例**:
```json
// JSON 配置文件
"filePath": "data/example/config/settings.json"

// 函数文件
"filePath": "data/example/functions/setup.mcfunction"

// 深层路径
"filePath": "data/example/worldgen/biome/forest.json"
```

**支持的文件类型**:
- `.json`: JSON 配置文件，支持嵌套路径
- `.mcfunction`: 函数文件，支持命令参数修改
- `.txt`: 文本文件（部分支持）

**注意事项**:
- 路径区分大小写
- 文件必须存在，否则配置无法应用
- 建议使用相对路径，不要使用绝对路径

---

#### 5. `locations` (可选)

**类型**: `Array<Object>`

**说明**: 批量修改配置，一个配置项可同时修改多个文件位置

**使用场景**: 当需要将一个配置应用到多个文件或多个位置时使用

**结构**:
```json
"locations": [
  {
    "filePath": "String",      // 目标文件路径
    "key": "String",           // 配置键
    "description": "String"    // 描述（可选）
  }
]
```

**示例**:
```json
{
  "debug_mode": {
    "type": "boolean",
    "default": "false",
    "locations": [
      {
        "filePath": "data/example/config/debug.json",
        "key": "debug.enabled",
        "description": "主调试配置"
      },
      {
        "filePath": "data/example/config/main.json",
        "key": "debug_mode",
        "description": "主配置中的调试开关"
      },
      {
        "filePath": "data/example/functions/debug.mcfunction",
        "key": "debug_enabled",
        "description": "调试函数控制"
      }
    ]
  }
}
```

**与 `filePath` + `key` 的区别**:
- `filePath` + `key`: 单个位置修改，简单配置
- `locations`: 多个位置修改，适用于全局开关、同步参数等场景

**注意事项**:
- `locations` 和 `filePath`+`key` 不能同时使用
- 数组中的每个对象都必须有 `filePath` 和 `key`
- `description` 是可选的，用于调试和日志记录

---

#### 6. `name` (可选)

**类型**: `Object` (多语言映射)

**说明**: 配置项在界面中显示的标题

**结构**:
```json
"name": {
  "zh_cn": "简体中文标题",
  "en_us": "English Title"
}
```

**示例**:
```json
"name": {
  "zh_cn": "生物生成率",
  "en_us": "Mob Spawn Rate"
}
```

**优先级**: 高于 `displayNameKey`

**如果不配置**: 显示配置项的键名（JSON 中的 key）

---

#### 7. `description` (可选)

**类型**: `Object` (多语言映射)

**说明**: 配置项下方的说明文本

**显示位置**: 配置项名称下方，使用灰色小字体显示

**结构**:
```json
"description": {
  "zh_cn": "简体中文描述",
  "en_us": "English Description"
}
```

**示例**:
```json
"description": {
  "zh_cn": "控制生物的生成频率，数值越大生成越快",
  "en_us": "Controls mob spawn frequency, higher values spawn faster"
}
```

**与 `tooltip` 的区别**:
- `description`: 始终显示在配置项下方
- `tooltip`: 鼠标悬停时显示

---

#### 8. `tooltip` (可选)

**类型**: `Object` (多语言映射)

**说明**: 鼠标悬停时显示的提示文本

**触发方式**: 鼠标悬停在配置项上

**结构**:
```json
"tooltip": {
  "zh_cn": "简体中文提示",
  "en_us": "English Tooltip"
}
```

**示例**:
```json
"tooltip": {
  "zh_cn": "拖动滑块调整数值，或点击输入精确值",
  "en_us": "Drag slider to adjust value, or click to input precise value"
}
```

**与 `description` 的区别**:
- `tooltip`: 悬停时显示，适合详细说明
- `description`: 始终显示，适合简短说明

**如果不配置**: 不显示提示

---

#### 9. `category` (可选)

**类型**: `String`

**说明**: 配置项所属的分类，支持多级分类

**格式**: 使用 `/` 分隔多级分类

**示例**:
```json
// 单级分类
"category": "game_settings"

// 多级分类
"category": "game_settings/difficulty_settings"

// 三级分类
"category": "game_settings/difficulty_settings/combat"
```

**界面显示效果**:
```
游戏设置
  └─ 难度设置
      └─ 战斗难度等级 [配置项]
```

**如果不配置**: 配置项显示在数据包根分类下

---

#### 10. `categoryDisplayNames` (可选)

**类型**: `Object` (多语言映射的多级映射)

**说明**: 自定义分类的显示名称

**结构**:
```json
"categoryDisplayNames": {
  "zh_cn": {
    "game_settings": "游戏设置",
    "difficulty_settings": "难度设置"
  },
  "en_us": {
    "game_settings": "Game Settings",
    "difficulty_settings": "Difficulty Settings"
  }
}
```

**使用场景**: 当分类的键名不够友好时，使用这个参数自定义显示名称

**如果不配置**: 显示分类的键名（category 中的字符串）

---

#### 11. `displayNameKey` (可选)

**类型**: `String`

**说明**: 引用数据包语言文件中的显示名称键

**使用场景**: 当数据包已经有语言文件时，避免重复定义文本

**示例**:
```json
"displayNameKey": "datapack.example.config.spawn_rate.name"
```

**对应语言文件** (`assets/example/lang/zh_cn.json`):
```json
{
  "datapack.example.config.spawn_rate.name": "生物生成率"
}
```

**优先级**: 低于 `name`

---

#### 12. `descriptionKey` (可选)

**类型**: `String`

**说明**: 引用数据包语言文件中的描述文本键

**使用场景**: 当数据包已经有语言文件时，避免重复定义文本

**示例**:
```json
"descriptionKey": "datapack.example.config.spawn_rate.desc"
```

**对应语言文件** (`assets/example/lang/zh_cn.json`):
```json
{
  "datapack.example.config.spawn_rate.desc": "控制生物的生成频率"
}
```

**优先级**: 低于 `description`

---

#### 13. `tooltipKey` (可选)

**类型**: `String`

**说明**: 引用数据包语言文件中的提示文本键

**使用场景**: 当数据包已经有语言文件时，避免重复定义文本

**示例**:
```json
"tooltipKey": "datapack.example.config.spawn_rate.tooltip"
```

**对应语言文件** (`assets/example/lang/zh_cn.json`):
```json

  "datapack.example.config.spawn_rate.tooltip": "数值范围: 0-100"
}
```

**优先级**: 低于 `tooltip`

### 特定类型参数

#### 14. `min` (slider, double_slider, int, double 必需)

**类型**: `Number`

**说明**: 数值的最小值（包含）

**示例**:
```json
"min": 0
"min": 0.1
```

---

#### 15. `max` (slider, double_slider, int, double 必需)

**类型**: `Number`

**说明**: 数值的最大值（包含）

**示例**:
```json
"max": 100
"max": 5.0
```

---

#### 16. `step` (slider, double_slider 可选)

**类型**: `Number`

**说明**: 滑块拖动的步长

**默认值**: `1` (slider) 或 `0.1` (double_slider)

**示例**:
```json
"step": 1      // 整数滑块，每次变化 1
"step": 0.1    // 浮点滑块，每次变化 0.1
```

---

#### 17. `enumValues` (enum, enum_selector, dropdown_menu 必需)

**类型**: `Array<String>`

**说明**: 枚举类型的可选值列表

**示例**:
```json
"enumValues": ["easy", "normal", "hard", "expert"]
```

**注意事项**:
- 数组中的值必须是字符串
- 这些值会存储在配置中（非显示名称）
- 建议使用中划线或下划线命名：`easy_mode`、`easy-mode`

---

#### 18. `enumDisplayNames` (enum, enum_selector 可选)

**类型**: `Object` (多语言映射)

**说明**: 枚举值的多语言显示名称

**结构**:
```json
"enumDisplayNames": {
  "zh_cn": {
    "easy": "简单",
    "normal": "普通"
  },
  "en_us": {
    "easy": "Easy",
    "normal": "Normal"
  }
}
```

**如果不配置**: 显示枚举值本身（如 `easy`, `normal`）

---

### 参数组合规则

#### 基本规则

1. **每个配置项必须包含**: `type` 和 `default`
2. **位置指定方式（二选一）**:
   - 方式 A: `filePath` + `key`
   - 方式 B: `locations` 数组
3. **文本相关（可选）**: `name`, `description`, `tooltip`
4. **分类相关（可选）**: `category`, `categoryDisplayNames`
5. **本地化（可选）**: `displayNameKey`, `descriptionKey`, `tooltipKey`

#### 类型特定规则

| 类型 | 必需参数 | 可选参数 |
|-----|---------|---------|
| slider | `min`, `max` | `step` |
| double_slider | `min`, `max` | `step` |
| int | `min`, `max` | - |
| double | `min`, `max` | - |
| boolean | - | - |
| string | - | - |
| enum | `enumValues` | `enumDisplayNames` |
| enum_selector | `enumValues` | `enumDisplayNames` |
| dropdown_menu | `enumValues` | - |
| color | - | - |

## 性能优化

### 1. 线程安全优化

- **线程安全容器**: 使用 `ConcurrentHashMap` 替代 `HashMap` 确保多线程环境下的数据安全
- **原子操作**: 配置项的读写操作通过线程安全容器保证一致性
- **无锁设计**: 避免使用同步锁，提高并发性能

**实现类**: `DatapackSettingsManager`

### 2. 延迟保存机制

- **延迟保存**: 引入 500ms 延迟保存机制，避免频繁的文件 I/O 操作
- **批量保存**: 多个配置修改在延迟期内合并为一次保存操作
- **定时任务**: 使用 `ScheduledExecutorService` 管理延迟保存任务

**优化效果**: 减少 70% 的文件写入操作，提升响应速度

### 3. 字符串操作优化

- **StringBuilder**: 使用 `StringBuilder` 替代字符串拼接，减少内存分配
- **缓存复用**: 对频繁使用的字符串进行缓存和复用
- **格式优化**: 优化颜色值格式化和解析逻辑

**实现类**: `DatapackConfigGuiBuilder`, `ColorPickerBuilder`

### 4. 缓存机制

- **配置缓存**: 解析后的配置信息缓存到内存
- **文件缓存**: 避免重复读取相同文件
- **数据包缓存**: 已扫描的数据包信息缓存
- **LRU 淘汰**: 实现基于访问频率的缓存淘汰策略

**实现类**: `CacheManager`

### 5. 代码复杂度优化

- **方法拆分**: 将复杂方法拆分为多个职责单一的小方法
- **输入验证**: 增强参数验证和错误处理
- **异常处理**: 改进异常处理机制，提供更清晰的错误信息
- **日志优化**: 优化日志级别和输出内容

**优化效果**: 提高代码可读性和可维护性，减少潜在错误

### 6. 性能监控

提供性能监控工具，帮助识别性能瓶颈。

**实现类**: `PerformanceMonitor`

**监控指标**:
- 配置解析时间
- 文件修改时间
- 界面构建时间
- 内存使用情况
- 线程安全状态

### 7. 优化建议

#### 数据包开发者

1. **减少配置项数量**: 只将需要用户修改的配置项放入配置界面
2. **使用批量修改**: 相同配置使用 `locations` 数组
3. **合理分类**: 避免过深的分类层级
4. **简化本地化**: 使用内嵌本地化而非引用语言文件

#### 玩家

1. **禁用不需要的数据包**: 在界面中关闭不需要的数据包
2. **定期清理**: 删除不使用的数据包
3. **避免频繁修改**: 集中修改配置项，减少文件操作

## 常见问题

### 1. 配置修改后没有生效

**可能原因**:
- 数据包未重新加载
- 配置文件路径错误
- 修改的文件不是数据包使用的文件

**解决方案**:
1. 确认数据包已重新加载（通常需要重新进入世界）
2. 检查 `datapack_config.json` 中的 `filePath` 是否正确
3. 查看游戏日志确认配置是否成功应用

### 2. 配置界面显示异常

**可能原因**:
- 配置文件格式错误
- 缺少必需的配置参数
- 本地化文本缺失

**解决方案**:
1. 检查 JSON 格式是否正确
2. 确认所有必需参数都已配置
3. 添加缺失的本地化文本

### 3. ZIP 数据包无法修改

**可能原因**:
- 文件权限不足
- 磁盘空间不足
- ZIP 文件损坏

**解决方案**:
1. 检查文件权限
2. 释放磁盘空间
3. 验证 ZIP 文件完整性
4. 查看 `.backup_*.zip` 备份文件

### 4. 批量修改功能不工作

**可能原因**:
- `locations` 数组格式错误
- 文件路径不正确
- JSON 路径不正确

**解决方案**:
1. 检查 `locations` 数组的格式
2. 确认所有文件路径都存在
3. 验证 JSON 路径格式（使用 `/` 分隔）

### 5. 嵌套路径不工作

**可能原因**:
- 路径分隔符错误
- JSON 结构不匹配
- 路径层级不存在

**解决方案**:
1. 使用 `/` 作为路径分隔符
2. 确认 JSON 文件结构与路径匹配
3. 检查中间层级是否存在

### 6. 本地化显示不正确

**可能原因**:
- 语言代码错误
- 本地化文本缺失
- 本地化优先级问题

**解决方案**:
1. 确认语言代码正确（`zh_cn` 或 `en_us`）
2. 检查本地化文本是否存在
3. 理解本地化优先级规则

### 7. 数据包扫描不到

**可能原因**:
- 数据包文件夹路径错误
- 数据包缺少 `datapack_config.json`
- 数据包格式不正确

**解决方案**:
1. 检查 Forge 配置中的数据包文件夹路径
2. 确认数据包根目录有 `datapack_config.json`
3. 验证数据包格式（ZIP ）

### 8. 性能问题

**可能原因**:
- 配置项过多
- 数据包数量过多
- 文件修改频繁

**解决方案**:
1. 减少配置项数量
2. 禁用不需要的数据包
3. 启用缓存机制

### C. API 文档

#### DatapackConfigParser

数据包配置解析器，负责解析 `datapack_config.json` 文件。

**主要方法**:
- `parseConfig(File file)`: 解析配置文件
- `parseMetadata(JsonObject metadata)`: 解析元数据
- `parseConfigEntries(JsonObject config)`: 解析配置项

#### DatapackFileModifier

数据包文件修改器，负责将配置应用到数据包文件。

**主要方法**:
- `applyDatapackConfig(File datapackFolder, Map<String, String> configChanges)`: 应用配置到数据包
- `applyDatapackConfigToZip(File zipFile, Map<String, String> configChanges)`: 应用配置到 ZIP 数据包
- `modifyJsonFile(File file, String key, String value)`: 修改 JSON 文件
- `modifyFunctionFile(File file, String key, String value)`: 修改函数文件

#### DatapackConfigGuiBuilder

数据包配置 GUI 构建器，负责动态生成配置界面。

**主要方法**:
- `buildDatapackConfigCategories(ConfigBuilder builder, ConfigEntryBuilder entryBuilder)`: 构建数据包配置分类
- `buildDatapackCategory(ConfigBuilder builder, ConfigEntryBuilder entryBuilder, DatapackSettings settings)`: 构建单个数据包分类
- `buildConfigEntry(ConfigEntryBuilder entryBuilder, String entryKey, ConfigMetadata metadata, Map<String, String> currentValues)`: 构建单个配置项
